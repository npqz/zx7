Lua Programming Basics

Variables and Data Types 🔗

Lua is dynamically typed, meaning variables don't need explicit type declarations.

-- Numbers local health = 100 local money = 1500.50 -- Strings local playerName = "John Doe" local message = 'Welcome to the server!' -- Booleans local isAlive = true local isAdmin = false -- Tables (arrays and objects) local weapons = {"pistol", "rifle", "shotgun"} local player = { name = "John", id = 12345, health = 100 }

Functions 🔗

Functions are first-class values in Lua and can be stored in variables.

-- Basic function function greetPlayer(name) return "Hello, " .. name .. "!" end -- Local function (recommended) local function calculateDamage(weapon, distance) local baseDamage = weapon.damage local damageReduction = distance * 0.1 return baseDamage - damageReduction end -- Anonymous function local healPlayer = function(playerId, amount) -- Heal player logic here print("Healed player " .. playerId .. " for " .. amount .. " HP") end

Control Structures 🔗

-- If statements if health <= 0 then print("Player is dead!") elseif health < 50 then print("Player needs medical attention") else print("Player is healthy") end -- For loops for i = 1, 10 do print("Count: " .. i) end -- Iterate through table local players = {"Alice", "Bob", "Charlie"} for index, name in ipairs(players) do print(index .. ": " .. name) end -- While loop local countdown = 5 while countdown > 0 do print("Starting in " .. countdown) countdown = countdown - 1 Wait(1000) -- FiveM function to wait 1 second end

Tables in Detail 🔗

Tables are Lua's only data structure and can represent arrays, objects, and more.

-- Array-like table local vehicles = {"adder", "zentorno", "t20"} -- Object-like table local playerData = { name = "John", money = 5000, vehicles = {"adder", "zentorno"}, position = {x = 100.0, y = 200.0, z = 30.0} } -- Adding elements playerData.bank = 10000 playerData["job"] = "police" -- Accessing nested data local playerX = playerData.position.x print("Player's X coordinate: " .. playerX)

FiveM Lua Development

Resource Structure 🔗

Every FiveM resource needs a proper folder structure and manifest file.

📁 Typical Resource Structure

my-resource/ ├── fxmanifest.lua ├── client.lua ├── server.lua └── shared.lua (optional)

FX Manifest File 🔗

The fxmanifest.lua file defines your resource configuration.

fx_version 'cerulean' game 'gta5' author 'Your Name' description 'My awesome FiveM resource' version '1.0.0' -- Scripts client_script 'client.lua' server_script 'server.lua' shared_script 'shared.lua' -- Dependencies dependencies { 'es_extended', -- ESX framework (if using) 'mysql-async' -- Database (if needed) }

Events System 🔗

FiveM uses an event-driven architecture for client-server communication.

Triggering Events

-- Client to Server TriggerServerEvent('my-resource:doSomething', data) -- Server to Client TriggerClientEvent('my-resource:updateUI', source, data) -- Server to All Clients TriggerClientEvent('my-resource:broadcast', -1, message)

Registering Event Handlers

-- On Client RegisterNetEvent('my-resource:updateUI') AddEventHandler('my-resource:updateUI', function(data) print('Received data from server: ' .. json.encode(data)) end) -- On Server RegisterNetEvent('my-resource:doSomething') AddEventHandler('my-resource:doSomething', function(data) local playerId = source print('Player ' .. playerId .. ' sent: ' .. json.encode(data)) end)

Building Custom Events

Create custom events for gameplay mechanics like quests or interactions.

📄 client.lua

-- Start quest RegisterCommand('startquest', function() TriggerServerEvent('quest:start', GetPlayerServerId(PlayerId())) end, false) -- Receive quest update RegisterNetEvent('quest:update') AddEventHandler('quest:update', function(questData) SetNotificationTextEntry('STRING') AddTextComponentString('Quest: ' .. questData.name .. ' - ' .. questData.status) DrawNotification(0, 1) end)

📄 server.lua

local quests = {} RegisterNetEvent('quest:start') AddEventHandler('quest:start', function(playerId) quests[playerId] = { name = "Find the Treasure", status = "Started" } TriggerClientEvent('quest:update', playerId, quests[playerId]) print("Player " .. playerId .. " started quest") end)

Native Functions 🔗

FiveM provides access to GTA V's native functions for game interaction.

-- Get player position local ped = PlayerPedId() local x, y, z = table.unpack(GetEntityCoords(ped, false)) -- Spawn a vehicle local vehicleHash = GetHashKey('adder') RequestModel(vehicleHash) while not HasModelLoaded(vehicleHash) do Wait(1) end local vehicle = CreateVehicle(vehicleHash, x, y, z, 0.0, true, false) SetPedIntoVehicle(ped, vehicle, -1)

Threads and Loops 🔗

Use Citizen.CreateThread for continuous operations.

-- Basic thread Citizen.CreateThread(function() while true do Wait(0) -- Important: prevents server freezing -- Your continuous code here local ped = PlayerPedId() local health = GetEntityHealth(ped) if health <= 0 then print("Player died!") break -- Exit the loop end end end) -- Timed thread (runs every 5 seconds) Citizen.CreateThread(function() while true do Wait(5000) -- 5 seconds print("This runs every 5 seconds") end end)

Performance Monitoring 🔗

Monitor your scripts to ensure they don't cause lag. Use profiling to measure execution time and optimize loops.

-- Client-side FPS monitoring Citizen.CreateThread(function() while true do Wait(1000) local fps = 1 / GetFrameTime() print("Current FPS: " .. math.floor(fps)) end end) -- Server-side profiling example local function profileFunction(func, name) local start = GetGameTimer() func() local duration = GetGameTimer() - start print("Function " .. name .. " took " .. duration .. "ms") end

Tip: Avoid tight loops with Wait(0) and cache frequently used natives.

Permission Systems 🔗

Restrict commands to specific roles like admin or moderator.

📄 server.lua

local permissions = { [1] = "admin", -- Player ID 1 is admin [2] = "moderator" -- Player ID 2 is moderator } RegisterCommand('kick', function(source, args, rawCommand) if permissions[source] == "admin" then local targetId = tonumber(args[1]) DropPlayer(targetId, "Kicked by admin") TriggerClientEvent('chat:addMessage', -1, { args = {"System", "Player " .. targetId .. " was kicked"} }) else TriggerClientEvent('chat:addMessage', source, { args = {"System", "Permission denied"} }) end end, true)

Practical Examples

Example 1: Simple Teleport Command 🔗

Create a command that teleports players to a specific location.

📄 client.lua

-- Register command RegisterCommand('tpls', function(source, args, rawCommand) local ped = PlayerPedId() -- Los Santos Airport coordinates local x, y, z = -1037.5, -2737.6, 20.1 -- Teleport player SetEntityCoords(ped, x, y, z, false, false, false, true) -- Notify player SetNotificationTextEntry('STRING') AddTextComponentString('Teleported to Los Santos Airport!') DrawNotification(0, 1) end, false)

Example 2: Vehicle Spawner 🔗

Spawn different vehicles with a command system.

📄 client.lua

local function spawnVehicle(vehicleName) local ped = PlayerPedId() local x, y, z = table.unpack(GetEntityCoords(ped, false)) local heading = GetEntityHeading(ped) -- Get vehicle hash local vehicleHash = GetHashKey(vehicleName) -- Check if model exists if not IsModelInCdimage(vehicleHash) or not IsModelAVehicle(vehicleHash) then SetNotificationTextEntry('STRING') AddTextComponentString('Invalid vehicle model: ' .. vehicleName) DrawNotification(0, 1) return end -- Request model RequestModel(vehicleHash) while not HasModelLoaded(vehicleHash) do Wait(1) end -- Spawn vehicle local vehicle = CreateVehicle(vehicleHash, x + 3, y, z, heading, true, false) SetVehicleOnGroundProperly(vehicle) -- Put player in vehicle TaskWarpPedIntoVehicle(ped, vehicle, -1) -- Set model as no longer needed SetModelAsNoLongerNeeded(vehicleHash) SetNotificationTextEntry('STRING') AddTextComponentString('Vehicle spawned: ' .. vehicleName) DrawNotification(0, 1) end -- Register commands for different vehicles RegisterCommand('car', function(source, args, rawCommand) local vehicleName = args[1] or 'adder' spawnVehicle(vehicleName) end, false) RegisterCommand('bike', function(source, args, rawCommand) spawnVehicle('sanchez') end, false) RegisterCommand('heli', function(source, args, rawCommand) spawnVehicle('buzzard2') end, false)

Example 3: Player Management System 🔗

Server-side player tracking and management.

📄 server.lua

local players = {} -- Player joined server AddEventHandler('playerConnecting', function(name, setKickReason, deferrals) local playerId = source players[playerId] = { name = name, joinTime = os.time(), money = 5000, -- Starting money level = 1 } print(name .. ' connected to the server') end) -- Player left server AddEventHandler('playerDropped', function(reason) local playerId = source local playerData = players[playerId] if playerData then print(playerData.name .. ' left the server. Reason: ' .. reason) players[playerId] = nil -- Clean up end end) -- Get player info command RegisterCommand('playerinfo', function(source, args, rawCommand) local playerId = tonumber(args[1]) or source local playerData = players[playerId] if playerData then local info = string.format( 'Player: %s | Money: $%d | Level: %d | Play Time: %d minutes', playerData.name, playerData.money, playerData.level, math.floor((os.time() - playerData.joinTime) / 60) ) TriggerClientEvent('chat:addMessage', source, { color = {255, 0, 0}, multiline = true, args = {"System", info} }) else TriggerClientEvent('chat:addMessage', source, { color = {255, 0, 0}, multiline = true, args = {"System", "Player not found!"} }) end end, true) -- true = server-side command

Example 4: Client-Server Communication 🔗

Money system with server validation.

📄 client.lua

-- Request money from server RegisterCommand('givemoney', function(source, args, rawCommand) local amount = tonumber(args[1]) or 100 TriggerServerEvent('money:request', amount) end, false) -- Receive money update from server RegisterNetEvent('money:update') AddEventHandler('money:update', function(newAmount) SetNotificationTextEntry('STRING') AddTextComponentString('Money updated: $' .. newAmount) DrawNotification(0, 1) end)

📄 server.lua

-- Handle money request RegisterNetEvent('money:request') AddEventHandler('money:request', function(amount) local playerId = source -- Validate amount if amount > 1000 then amount = 1000 -- Cap at 1000 end -- Update player money (in real scenario, use database) if players[playerId] then players[playerId].money = players[playerId].money + amount -- Send updated amount back to client TriggerClientEvent('money:update', playerId, players[playerId].money) print('Gave $' .. amount .. ' to player ' .. playerId) end end)

Create styled chat messages and commands for better player interaction.

RegisterCommand('ooc', function(source, args, rawCommand) local message = table.concat(args, " ") TriggerServerEvent('chat:ooc', message) end, false) RegisterNetEvent('chat:oocMessage') AddEventHandler('chat:oocMessage', function(playerName, message) TriggerEvent('chat:addMessage', { color = {100, 100, 255}, args = {"[OOC] " .. playerName, message} }) end)
